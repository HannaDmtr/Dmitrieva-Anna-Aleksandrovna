Тег picture

С появлением устройств, имеющих разные размеры экрана, разрешения и сложные требования, начали возникать вопросы об отзывчивости тега img и возможности его использования в приложениях, рассчитанных на множество различных устройств. Основными вопросами стали: смена разрешения и ориентация графики (проблема отображения различных изображений при разных размерах экрана). Поэтому для улучшения производительности сайтов и удобства пользователя разработчики стали активно использовать тег picture.

У тега picture есть атрибуты srcset и sizes. Атрибут srcset может принимать несколько изображений с соответствующей шириной в пикселях, а браузер использует эти значения для выбора между указанными изображениями. В показанном ниже примере есть три версии одного и того же изображения в трёх разных размерах. Атрибут sizes задаёт пространство, которое изображение будет занимать на экране.

Ориентация графики при помощи атрибута media

В большинстве случаев, изображение замечательно выглядящее на больших экранах, оказывается обрезанным или очень мелким при просмотре на мобильном устройстве. Эта проблема решается созданием различных версий изображения для разных размеров экрана. Эти версии могут быть альбомными, портретными или любыми другими изменёнными версиями того же изображения.

С помощью тега picture мы можем легко обеспечить смену разрешения, воспользовавшись несколькими тегами source внутри тега picture. Затем можно использовать атрибут media для задания различных условий среды. Также можно использовать атрибуты srcset и sizes. В показанном ниже примере демонстрируется полная реализация ориентации графики и смены разрешения при помощи тега picture.

Если экран находится в альбомной ориентации, то браузер будет отображать изображения из первого набора, а если в портретной, то из второго набора. Также можно использовать атрибут media с параметрами max-width и min-width.

Последний тег img используется для обратной совместимости с браузерами, не поддерживающими теги picture.

Использование с частично поддерживаемыми типами изображений

У некоторых браузеров существуют ограничения в обработке современных типов изображений (таких как webp, svg и avif), и если не использовать совместмые типы изображений, это может вызвать проблемы. Тег picture помогает решить эту проблему, т.к. он позволяет указать внутри несколько источников.

В показанный ниже пример включены три типа изображений в форматах avif, webp, svg и jpg. Сначала браузер попробует формат avif, если не получится, то попробует webp, затем svg. Если браузер не поддерживает ни один из них, то использует изображение jpg.
